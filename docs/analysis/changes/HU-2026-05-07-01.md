# HU-2026-05-07-01 — Reestructuración de Sub-recursos de JobProfile

## 1. Resumen ejecutivo

Reestructuración de la API de Perfiles de Puesto para adoptar un diseño RESTful basado en **sub-recursos unitarios**. Todas las colecciones embebidas (`Requirements`, `Functions`, `Relations`, `Competencies`, `Trainings`, `Benefits`, `WorkingConditions`, `DependentPositions`) se migran a endpoints independientes con operaciones `GET`, `POST`, `PUT`, `DELETE` por ítem.

- **Módulo:** Job Profiles / Competency Framework / Core API
- **Tipo:** Refactoring de API + Cambio de contrato + Schema migration
- **Estado:** En implementación
- **Fecha de inicio:** 2026-05-07

---

## 2. Motivación

El diseño anterior embebía todas las colecciones del perfil en el payload de creación (`POST`) y actualización (`PUT`) del perfil principal. Esto:

- Impedía operaciones granulares (agregar/editar/eliminar un solo ítem sin reenviar todo el arreglo).
- Generaba payloads masivos y frágiles a errores de concurrencia.
- No permitía distinguir el ciclo de vida de los sub-recursos del del perfil principal.

---

## 3. Cambios implementados

### 3.1 Domain Layer — `CLARIHR.Domain`

#### Entidades modificadas

Se agregaron `ConcurrencyToken` + método `Update(...)` a 7 sub-entidades del dominio de Job Profiles. La propiedad `PublicId` **ya existía heredada** de la clase base `Entity`:

| Entidad | ConcurrencyToken | `Update()` | Observaciones |
|---|---|---|---|
| `JobProfileRequirement` | ✅ Agregado | ✅ Agregado | `PublicId` heredado de `Entity` |
| `JobProfileFunction` | ✅ Agregado | ✅ Agregado | `PublicId` heredado de `Entity` |
| `JobProfileRelation` | ✅ Agregado | ✅ Agregado | `PublicId` heredado de `Entity` |
| `JobProfileTraining` | ✅ Agregado | ✅ Agregado | `PublicId` heredado de `Entity` |
| `JobProfileBenefit` | ✅ Agregado | ✅ Agregado | `PublicId` heredado de `Entity` |
| `JobProfileWorkingCondition` | ✅ Agregado | ✅ Agregado | `PublicId` heredado de `Entity` |
| `JobProfileDependentPosition` | ✅ Agregado | ✅ Agregado | `PublicId` heredado de `Entity` |
| `JobProfileCompetencyExpectation` | Ya existía | ✅ Agregado | En `CompetencyFramework` namespace |

#### Error corregido — CS0108

Durante la implementación inicial se redeclaró `PublicId` en las 7 entidades causando `CS0108: hides inherited member Entity.PublicId`. Se corrigió eliminando la redeclaración ya que la base `Entity` ya provee `public Guid PublicId { get; protected set; }`.

## 6. Breaking changes

- Los payloads de `POST /job-profiles` y `PUT /job-profiles/{publicId}` dejarán de aceptar las propiedades de colección (`requirements`, `functions`, etc.).
- El front-end debe migrar a un flujo de guardado paso a paso: **primero el perfil base**, luego los sub-recursos.
