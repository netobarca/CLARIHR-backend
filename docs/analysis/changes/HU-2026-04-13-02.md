# HU-2026-04-13-02 — Preview de validacion previa a finalize de expediente

## 1. Informacion general

- **Codigo HU:** HU-2026-04-13-02
- **Titulo:** Preview de validacion previa a finalize de expediente
- **Modulo:** Personnel Files / Employment
- **Fecha de cierre:** 2026-04-13
- **Estado:** Implementada
- **Responsable:** Codex
- **Referencia tecnica:** `src/CLARIHR.Application/Features/PersonnelFiles/FinalizePersonnelFile.cs`

## 2. Objetivo de la HU

Exponer un endpoint para que el frontend valide, antes de `finalize`, si el expediente cumple las reglas bloqueantes actuales de finalizacion.

## 3. Alcance implementado

- Nuevo endpoint `GET /api/v1/personnel-files/{id}/finalize/preview`.
- Parametro query `createUserAccount` opcional (default `true`).
- Response `FinalizePersonnelFilePreviewResponse` con `isEligible` e `issues` bloqueantes (`code`, `message`, `section`, `fieldKey`, `isBlocking`).
- Reuso de logica de elegibilidad entre `finalize` y `finalize/preview` mediante `FinalizePersonnelFileValidationResolver`.

## 4. Reglas funcionales cubiertas por preview

- Expediente `Employee`.
- Estado `Draft` y sin usuario ya vinculado.
- `InstitutionalEmail` requerido.
- `AssignedPositionSlotPublicId` requerido y plaza existente.
- Si `createUserAccount = true`: plaza con rol valido.
- Si `createUserAccount = true`: no conflicto por correo institucional vinculado a otro expediente.

## 5. Pruebas

- Se agregan pruebas unitarias para:
  - `PreviewFinalizePersonnelFileQueryHandler` con issues de correo faltante y rol faltante.
  - Camino elegible cuando `createUserAccount = false` aunque falte rol de plaza.
  - Regresion de `FinalizePersonnelFileCommandHandler` para mantener rechazo por rol faltante cuando `createUserAccount = true`.
  - Default `createUserAccount = true` en controller para `finalize` y `finalize/preview`.
