## 2. Objetivo de la HU

Sacar exportes grandes del request path, limitar exportes sincronos, centralizar generacion de archivos, agregar pipeline asincrono persistido y optimizar lecturas pesadas de reporting.

## 3. Alcance implementado

### Incluye
- `ReportPerformanceOptions` con limites de export, batches, leases, intentos, retencion y limite de nodos.
- Limite sincrono de `5,000` filas con overflow `413 REPORT_EXPORT_TOO_LARGE`.
- Entidad `ReportExportJob`, repositorio, worker `BackgroundService`, storage Azure Blob y endpoints de creacion/listado/estado/descarga/cancelacion.
- Writer comun `ReportExportFileWriter` para CSV/XLSX/JSON y auditoria comun de exportes.
- Recursos cubiertos: `PERSONNEL_FILES`, `PERSONNEL_FILE_PERSONNEL_ACTIONS`, `PERSONNEL_FILE_PAYROLL_TRANSACTIONS`, `ORG_UNITS`, `POSITION_SLOTS`, `SALARY_TABULATOR`, `COST_CENTERS`, `LEGAL_REPRESENTATIVES` y `JOB_PROFILE_COMPETENCY_MATRIX`.
- Analytics de expedientes con agregaciones SQL y detalle de expediente con `AsSplitQuery()`.
- Migracion EF para `report_export_jobs` e indices de performance en `personnel_files`.

### No incluye
- Hangfire, Redis o cola externa.
- Antivirus o storage documental por streaming.
- Eliminacion del export single-resource de `JOB_PROFILE`.
- Historial de jobs previo, porque no existia persistencia historica.

## 4. Impacto funcional

Los usuarios pueden seguir usando exportes existentes para datasets pequenos. Cuando el resultado supera el limite sincrono, la API responde `413` y el cliente debe crear un job asincrono, consultar su estado y descargarlo cuando este `Succeeded`.

### Flujo afectado
- Reportes y exportes de RRHH.
- Exportes de estructura, plazas, tabulador, centros de costo, representantes legales y matriz de competencias.

## 12. Riesgos, limitaciones y pendientes

### Limitaciones actuales
- El pipeline asincrono requiere Blob Storage configurado; sin storage responde `503`.
- Los artefactos expiran a las `24` horas.
- No se agrego cola externa; la coordinacion se hace con leases en PostgreSQL.

### Pendientes
- Integration test end-to-end del pipeline asincrono con storage fake o emulado.
- Limpieza fisica de blobs vencidos si el volumen de jobs crece.
