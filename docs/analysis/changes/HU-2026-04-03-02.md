# HU-2026-04-03-02 — Cierre de implementación

## 1. Información general

- **Código HU:** HU-2026-04-03-02
- **Título:** Estandarización de respuestas de error para model binding y JSON inválido
- **Módulo:** Core API / Platform Backoffice API / Error Handling
- **Fecha de cierre:** 2026-04-03
- **Estado:** Completada
- **Responsable:** Codex
- **Referencia funcional:** Requerimiento para unificar la respuesta de error al guardar `Job Profiles` y alinear validaciones HTTP con el contrato estándar del backend
- **Referencia técnica:** `ProblemDetailsFactory`, `Program`, `JobProfilesController`, `ApiIntegrationTests`

---

## 2. Objetivo de la HU

### Resumen
Unificar las respuestas `400` generadas por `model binding` o body JSON inválido con el mismo contrato `ProblemDetails` que ya usan los errores de negocio, de forma que frontend y soporte reciban `code`, `traceId` y `errors` consistentes.

---

## 3. Alcance implementado

### Incluye
- Intercepción explícita de `ModelState` inválido en Core API y Backoffice API mediante `ApiBehaviorOptions.InvalidModelStateResponseFactory`.
- Conversión de errores automáticos de ASP.NET a `common.validation` usando el `ProblemDetailsFactory` propio del proyecto.
- Normalización de keys de validación para remover prefijos internos como `request` y `$.`.
- Saneamiento de mensajes de conversión JSON para devolver mensajes públicos como UUID inválido en lugar de detalles crudos del parser.
- Cobertura de integración dirigida en `JobProfiles` para el caso reportado de `dependentJobProfilePublicId` inválido.

### No incluye
- Rediseño de códigos de negocio existentes fuera del estándar actual `ProblemDetails + code`.
- Cambios a errores framework ajenos al pipeline de validación de requests de la aplicación.

---

## 4. Impacto funcional

### Cambios funcionales
- El frontend recibe el mismo envelope de error tanto para validaciones de negocio como para errores de body/request inválido.
- Los errores de campos inválidos en `JobProfiles` ahora usan keys del contrato público y mensajes legibles.

### Flujo afectado
- Guardado de `Job Profiles`
- Cualquier endpoint Core API o Backoffice API que falle en `model binding` o deserialización JSON
