# HU-2026-04-13-01 — Finalizacion de expediente con creacion de cuenta opcional

## 1. Informacion general

- **Codigo HU:** HU-2026-04-13-01
- **Titulo:** Finalizacion de expediente con creacion de cuenta opcional
- **Modulo:** Personnel Files / Company Users
- **Fecha de cierre:** 2026-04-13
- **Estado:** Implementada
- **Responsable:** Codex
- **Referencia tecnica:** `src/CLARIHR.Application/Features/PersonnelFiles/FinalizePersonnelFile.cs`

## 2. Objetivo de la HU

Permitir que `finalize` en expedientes de personal decida explicitamente si debe aprovisionar cuenta de usuario o no, manteniendo compatibilidad con clientes actuales.

## 3. Alcance implementado

- Se agrega `createUserAccount` (nullable) al request `FinalizePersonnelFileRequest`.
- Si el flag viene omitido, el backend usa default `true` para preservar el comportamiento actual.
- `FinalizePersonnelFileResponse.User` pasa a nullable para soportar finalizaciones sin aprovisionamiento.
- `FinalizePersonnelFileCommandHandler` implementa dos rutas:
  - `createUserAccount = true`: flujo actual de aprovisionamiento, invitacion y vinculacion.
  - `createUserAccount = false`: finaliza expediente sin crear/reutilizar usuario.
- El agregado `PersonnelFile` agrega `CompleteWithoutLinkedUser()` para representar la finalizacion sin cuenta vinculada.

## 4. Reglas funcionales resultantes

- `finalize` sigue exigiendo expediente `Employee` en `Draft`, `InstitutionalEmail`, `AssignedPositionSlotId` y `ConcurrencyToken` valido.
- La validacion de rol IAM en la plaza solo aplica cuando `createUserAccount = true`.
- Al finalizar sin cuenta, `LinkedUserId` queda `null`.

## 5. Pruebas realizadas

- Nuevas unit tests en `FinalizePersonnelFileTests` para:
  - finalizacion sin cuenta y sin llamada a provisioning
  - rechazo por falta de correo institucional al finalizar sin cuenta
  - preservacion del flujo de aprovisionamiento cuando `createUserAccount = true`
- Nueva prueba en `PersonnelFileEmploymentControllerTests` para default `createUserAccount = true` cuando el campo se omite.
- Se actualiza `PersonnelFileDomainTests` con cobertura de `CompleteWithoutLinkedUser()`.

## 7. Riesgos o pendientes

- Este cambio no agrega un flujo para crear y vincular cuenta despues de finalizar sin aprovisionamiento; ese comportamiento queda cubierto por la creacion manual de usuarios de compania y una HU futura para vinculo explicito desde expediente.
