## Protocolo de uso (para el loop y para cada sesión nueva)

1. **Al iniciar** una sesión o iteración del loop: leer este archivo. El requerimiento activo es el primero en 🟡; si no hay ninguno, tomar el primero 🔴 (orden = prioridad).
2. Ejecutar la **"Próxima acción"** del requerimiento activo (es autocontenida: no depende de recordar la conversación).
3. **Al cerrar cada unidad de trabajo** (un PR completado con suite verde): marcar el checkbox, actualizar "Próxima acción" y agregar una línea a la **bitácora** con fecha y resultado (incluir hash de commit / número de PR si existe).
4. Un requerimiento pasa a 🟢 solo cuando: todos sus PRs están mergeados, la suite completa está verde, `openapi.yaml` sin drift, guía FE publicada y los pendientes de despliegue quedaron documentados/ejecutados.
5. Si algo bloquea (falta insumo del cliente, decisión de negocio, infraestructura): estado ⏸️ con el motivo y qué se necesita para destrabar; pasar al siguiente requerimiento 🔴 si lo hay.
6. Los requerimientos nuevos se agregan al final con la **plantilla** de la última sección (el orden en el archivo define la prioridad; se puede reordenar).
7. Cada PR se desarrolla contra la rama del requerimiento; convención de commits del repo: **sin trailer de co-autoría de IA**.
